复杂度度量

算法的复杂度从时间与空间两个维度进行度量，由于存在机器，输入数据等的差异，以算法实际运行时间难以作为算法之间比较标准，故在图灵机与随机存储模型下，将算法中的每一个步骤都可以分解为基本操作，基本操作可在常数时间内完成，故算法的复杂度就可以由基本操作的总个数导出。

渐进复杂度

定义算法时间复杂度为T(n)，由算法所有语句执行次数所决定。由于在小规模问题时，算法之间的效率差异较小，所以我们往往关注在更大规模情况下，时间复杂度变化的趋势和增长的速度，即渐进分析。

O

定义：
>
对于 n >> 2 有：
T(n) <= cf(n)，则认为f(n)为T(n)增长的上界，记为T(n)=O(f(n))

该符号确定了算法的复杂度上界，即对于规模为n的任意输入，运行时间均不超过O(f(n))，是对算法的保守估计。
在该记号下，函数的系数以及低次项将会被忽略，仅保留最高次项。

Ω

定义：
>

对于 n >> 2 有：
T(n) >= cg(n)，则认为g(n)为T(n)渐进下界，记为T(n)= Ω(g(n))

该记号是对算法的乐观估计，即对于任意规模n的输入，运行时间均不低于Ω(g(n))

Θ

上述两个已经描述了算法运行时间的区间，有种特殊情况，即若f(n)=g(n)时，就用Θ表示。
定义：

对于 n >> 2 有：c1h(n) <= T(n) <= c2h(n),即n足够大时，h(n)给出了T(n)的一个确界，记为 T(n) = Θ(h(n))

该符号给出算法复杂度的准确估计。

空间复杂度

衡量算法运行时的空间开销，上述符号同样适用于空间分析，有一点需要特别注意的，由于每个基本操作所设计的空间也为常数，所以时间复杂度本身即为空间复杂度的一个天然上界。
